Architecture

Five modules, one workspace

Each class in nexkore.py owns one responsibility and writes into the shared session directory.

ModuleResponsibilityKey output
EnvManagerInstalls and verifies Go, pip, and APT tool dependencies; creates isolated venvsTool status table
WorkspaceManagerBuilds the per-target, per-session directory tree and metadata.jsonSession workspace
ReconEngineRuns subfinder, dnsx, httpx, gau, katana, paramspider, naabu, git-exposure checks01_recon/, 03_secrets/
VulnScannerRuns nuclei, ffuf, dalfox, sqlmap, trufflehog, CORS and host-header checks02_vuln/, 03_secrets/

Get started

Running in under a minute

Requires Python 3.8+ and Go 1.21+ for the Go-based tools.

shellbash
# clone the repository
git clone https://github.com/SKumar8080/NEXKORE.git
cd nexkore

# install python dependencies
pip install -r requirements.txt

# launch
python3 nexkore.py
1

Clone and install

Pull the repo and install the three core Python dependencies — rich, pyfiglet, requests.

2

Set up your environment

Select option 1 in the menu to check, install, or update every Go / pip / apt-based scanning tool.

3

Create a workspace

Select option 2, enter your authorized target domain, and NEXKORE builds the full session directory tree.

4

Run and report

Trigger the full pipeline or individual modules, then generate a Markdown report from option 15.

!

Authorized use only

NEXKORE automates and organizes output from publicly available security tools — it does not contain exploit code of its own. Use it only on systems you own or have explicit written authorization to test. Unauthorized use is illegal in most jurisdictions, and you are solely responsible for how you use this tool.
